Acceptance Criteria

  • (sto16) when user selects an anatomy term from the dropdown, the query is by that term's ID, rather than by its text.
    • Example: search for 'limb'. Results should only include those for descendants of EMAPA:16405 (limb). It shoud NOT include things like "thin descending limb of outer medulla" (under renal system, loop of Henle)
  • (sto16) Cre search should NOT change (i.e., it should still search based on text tokens.)
  • (sto58) two rows displayed: A and B
  • (sto58) B only displayed if there are negative results returned
  • (sto58) (Assuming query semantics are correct) B row should be all red (or some white)
  • (sto58) if no negative result for B, B is not displayed
  • (sto101) for differential queries involving structures, show A and B (if there are results), but do not open either one.
  • (sto203) none

Functional Requirements

  • GXD Query Form has a second tab for the differential query
  • Differential Query Form has 3 ribbons: Structures only (A but not B), Stages only (I but not J), and Structures + Stages (A at I but not B at J).
  • Each ribbon has a submit and reset button. User is able to submit parameters from only one ribbon.
  • Has autocomplete boxes for structure A and another for structure B which use the EMAPA vocabulary.
  • Autocomplete is forced-select (user must pick from one of the values displayed). Query submits the EMAPA ID for the selected term. (NEW: sto16, May 2014)
  • Allows user to select multiple stages for both I and J
  • Allow user to specify J as a "no other stages" checkbox (equivalent to manually selecting all the other stages not in I)

Query functionality

  • Query excludes "ambiguous" and "not specified" results.
  • Differential queries include only "wild type" results in the query logic and in the displayed results sets. "Wild type" is defined as assays which are not associated with any allele pairs, or in situ reporter (knock-in) assays associated with a single allele pair which is heterozygous w.t. for the assayed gene.
  • Query converts EMAPA terms from the Structure autocomplete to EMAPS terms to submit the query. (NEW: sto16, May 2014)
  • Query returns genes that have positive expression in the A structure or any descendant terms in any of the I stages, and subtracts any genes that have positive expression in the B structure or any descendant terms in any of the J stages
  • Descendant terms are defined as following any path toward a leaf node in the DAG: these may be IS_A or PART_OF relationships.

Results Summary functionality

  • For those genes that satisfy the criteria, show the positive results from A and its descendants from the I stages
  • For those genes that satisfy the criteria, show negative/absent results, if any, for structure B for the J stages. This includes just the negative results in B. (No ancestors or descendants needed.)
  • Do not display "ambiguous" or "not specified" results.
  • For differential queries, the Genes tab is opened in the Results Summary by default
  • Differential Query Form is tied into history manager by opening the appropriate tab+ribbon if the query string represents a differential query.

Matrix Displays functionality

  • Display one row for structure A. If there are annotated results for descendant terms, the row is expandable.
  • Unlike the standard queries, the row for structure A is NOT expanded by default. (It does NOT display first-level child terms by default.) (sto101)
  • If negative results exist, display one row for structure B, below the row for structure A. This row is not expandable (as no descendant terms should be returned via rules above).
  • If structure B is a descendant term of structure A, then row(s) for B will ALSO appear in DAG location(s) beneath A when expanded.
  • No rows are displayed for descendant terms for B.
  • If NO structures are specified in the query, display all "default" rows having data for the specified Theiler Stages in their expanded or collapsed state, as with the Standard Query.
  • If NO stages are specified in the query, display Stage columns for every stage having data for structure A or structure B.
  • In Theiler Stage columns, if stages ARE specified in the query, display Stage columns only for the I and J stage(s).
  • (Theiler Stage columns should match the stages listed in the Theiler Stages filter.)
  • In Gene columns, display a column for each gene having a positive result for structure A or its descendants (in the I stage(s)), and for any additional genes having a negative result for structure B (in the J stage(s)).
  • (Gene columns should match the genes listed on the Genes tab.)
  • Color roll-up logic, modified from Standard Query functional requirements: (sto39)
    1. If there are any results under the cell where detected=true, then color=BLUE.
    2. Else, if the structure for THAT row (not looking at descendants) has result(s) that are NS or Ambiguous, then color=GRAY.
    3. Else, if the structure for THAT row (not looking at descendants) has result(s) that are Absent, then color=RED.
    4. Else, if there are any results under that cell where detected is Ambiguous, NS or Absent, color=GOLD.
  • Color shading of cells corresponds to number of positive or negative results for the row and column intersection.
  • Pop-up panel displays correct count of positive or negative results for the row and column intersection.
