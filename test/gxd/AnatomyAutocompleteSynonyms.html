Synonyms-handling in Anatomy Automplete List

  • User story 48, Change anatomy autocomplete handling of synonyms

Acceptance Criteria

  • For synonyms in the autocomplete, display the text of the synonym and "[synonym]" after it
  • Do this if it is only a synonym. If the text is a real term somewhere in the anatomy and a synonym elsewhere, don't display "[synonym]"
  • If the user selects a synonym in the autocomplete, display the synonym in the query field and search for terms/synonyms that contain all the tokens from that synonym. (Do not display the text "[synonym]".)

Tests

Skip to Manual Tests | Automated Tests

Manual tests

  1. Text in the autocomplete list that corresponds to one or more synonyms but no primary terms should display [synonym] next to the text.

    new test

    1. Open the GXD Query Form.
    2. In the Anatomical Structures field, enter heart muscle.
    3. Verify that [synonym] appears next to the text "heart muscle".
  2. Text in the autocomplete list that corresponds to one or more synonyms and also one or more primary terms should NOT display [synonym] next to the text.

    new test

    1. Open the GXD Query Form.
    2. In the Anatomical Structures field, enter kidney.
    3. Verify that [synonym] DOES NOT appear next to the text "kidney". (A primary term in TS 28 but a synonym in other Stages.)
  3. Text in the autocomplete list that corresponds to one or more primary terms should NOT display [synonym] next to the text.

    new test

    1. Open the GXD Query Form.
    2. In the Anatomical Structures field, enter cardiac muscle.
    3. Verify that [synonym] DOES NOT appear next to the text "cardiac muscle".
