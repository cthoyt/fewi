Anatomy Term Search

  • User story 10, Build autocomplete for Anatomy term search
  • User story 11, Search for expression results by Anatomy term
  • User story 88, Implement stop word list to anatomy search
  • User story 48, Change anatomy autocomplete handling of synonyms

Acceptance Criteria

  • Story 10: input text causes a dropdown list to appear with matching structure and synonyms
  • Story 10: when the user selects a term, the input box is populated with the selected text
  • Story 10: sorting is by score (exact matches first), then by alpha
  • Story 10: support string tokens (i.e. ""le lo"" includes ""left lobe"")
  • Story 10: does not support selecting multiple Structure Names
  • Story 11: user may select only one option from autocomplete list, and this text may be a Term or a Synonym text string from the AD
  • Story 11: selected input text tokens are matched against Structure Name (ANDing tokens)
  • Story 11: selected input text tokens are also matched against Synonyms (ANDing tokens) in order to find associated Structure Names
  • Story 11: all expression results associated with all matching Structure Names and their child structures are returned
  • Story 11: other parameters AND'ed to this
  • Story 11: reset button clears the widget
  • Story 11: update display user's query parameters
  • Story 11: update user help
  • Story 11: DO NOT update the autocomplete handling of synonyms to be like the vocab autocomplete
    • for each synonym, DO NOT display the structure name in square brackets
  • Story 11: the autocomplete will continue to display both Terms and Synonyms, undistinguished from one another, which match to the user's typed text
  • Story 88: Employ a Stop Words list (and,from,of,or,the,their,to)(as of 7/06/12)
  • Story 88: DO NOT perform stemming
  • Story 88: treat special characters (ampersand, at symbol, hyphen, apostrophe, slash, etc.) as if they were whitespace
  • Story 48: For synonyms in the autocomplete, display the text of the synonym and "[synonym]" after it
  • Story 48: Do this if it is only a synonym. If the text is a real term somewhere in the anatomy and a synonym elsewhere, don't display "[synonym]"
  • Story 48: If the user selects a synonym in the autocomplete, display the synonym in the query field and search for terms/synonyms that contain all the tokens from that synonym. (Do not display the text "[synonym]".)
