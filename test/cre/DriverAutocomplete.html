Cre Query Form - Driver Autocomplete Tests

  • Project Blizzard User story 48, Driver pick List becomes an Autocomplete list
  • Project Blizzard User story 48, Driver Autocomplete conforms to system standards

Acceptance Criteria

  • sto48: driver input field becomes a type-in that provides autocomplete functionality
  • sto48: autocomplete list is the list of drivers that match what the user is typing
  • sto48: wildcard support not necessary; query is always about 0 or 1 driver

  1. Verify that if no driver is selected, all results are returned

    1. Open the CRE query form
    2. Leave the Recombinase driven by field blank and click on Search
    3. Verify all driver results are returned
  2. Verify that autocomplete begins working on the 1st character

    jeffc 11/17/2015

    1. Open the CRE query form
    2. Type w in the recombinase driven by field
    3. Verify the autocomplete list appears after the "w" is entered
    4. Clear the recombinase driven by field
    5. Type W in the recombinase driven by field
    6. Verify the autocomplete list appears after the "W" is entered
  3. Verify that text copied-and-pasted into the driver input box returns proper results

    jeffc 11/17/2015

    1. Open the CRE query form
    2. Copy "acan", then paste it into the recombinase driven by field, hit the space bar
    3. Verify that the autocomplete functions and that the matching driver is displayed
  4. Verify that the browser's native "autofill" functionality is disabled by this Auto-Complete functionality

    klf 11/17/2015

    1. Open the CRE query form
    2. Confirm the browser's "autofill" functionality is turned on
    3. Enter some text in a different input text box (e.g., another query form like the Quick Search) - for example, "mgi:104876" - and Search.
    4. Clear the field and verify that entering the first few characters of the text from the previous step to reappear as a suggestion
    5. Enter the same first few characters in the recombinase driven by field and verify that the text from the previous step does not reappear as a suggestion
    6. Repeat these steps for supported browsers, versions, and OSes.
  5. Verify long lists scroll properly

    klf 11/17/2015

    1. Open the CRE query form
    2. Type T in the reocombinase driven by field and scroll back and forth through the auto list
    3. Verify the user can scroll up and down the list using a mouse or keyboard interaction
    4. Verify the last value in the list is "Tyrp1".
  6. Verify long lists do not get cut off by some other piece of the UI

    klf 11/17/2015

    3. Verify the full dimensions of the drop-down list are visible
