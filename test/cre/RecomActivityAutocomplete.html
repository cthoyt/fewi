Recombinase Activity Anatomy Autocomplete Tests

  • User story 126, Build autocomplete (prototype) for Recombinase Activity Anatomy term search

Acceptance Criteria

  • Story 126: the prototype presents a type in box for tissue/anatomy term
  • Story 126: type ahead implemented
  • Story 126: synonyms indicated by "syn." prefix
  • Story 126: zero vs non-zero result counts distinguished somehow ("+"/"-" or grayed text or...)
  • Story 126: don't worry about moving 0 counts down yet
  • Story 126: don't worry about showing actual counts yet
  • Story 126: stakeholders settle on main set of acceptable requirements:
    • Story 126: For synonyms in the autocomplete, display the text of the synonym and "[syn. of ...]" after it
    • Story 126: Do this if it is only a synonym. If the text is a real term somewhere in the anatomy and a synonym elsewhere, don't display "[syn. of ...]"
    • Story 126: The search and sort of this autocomplete mirrors what the GXD anatomical structures autocomplete does...
    • From GXD Anatomy autocomplete: input text causes a dropdown list to appear with matching structure and synonyms
    • From GXD Anatomy autocomplete: when the user selects a term, the input box is populated with the selected text
    • From GXD Anatomy autocomplete: sorting is by by score (exact matches first), then by alpha
    • From GXD Anatomy autocomplete: support string tokens (i.e. "le lo" includes "left lung lobar bronchus")
    • From GXD Anatomy autocomplete: does not support selecting multiple Structure Names
    • From GXD Anatomy autocomplete: If the user selects a synonym in the autocomplete, display the synonym in the query field and search for terms/synonyms that contain all the tokens from that synonym. (Do not display the text "[synonym]".)
