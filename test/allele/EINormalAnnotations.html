EI allow normal annotations to any MP term, not just headers

  • User story 73, Allow Normal annotations to any MP term, not just header terms

Acceptance Criteria

  • The "norm" qualifer can be applied to any MP term, not just header terms.
  • existing normal annotations migrated to new representation.

See also: PhenotypeAnnotLoader for tests of the load script.

Tests

Skip to Manual Tests | Automated Tests

Summary of Tests

  • Confirm you can add a normal qualifier to a new MP annotation
  • Confirm you can modify an existing non-load annotation sub term with a normal qualifier
  • Confirm you can modify an existing sanger load sub term with a normal qualifier
  • Confirm you can remove an existing normal qualifier from a non-load annotation term
  • Confirm you can remove an existing normal qualifier from a sanger load annotation term
  • Confirm all header terms default to normal qualifier
  • Confirm you can't change a header term to null qualifier

Manual tests

Note: Concordion doesn't like < and > signs directly in the HTML code. Encode them as "&lt;" and "&gt;", respectively.

  1. Verify you can add a normal qualifier to a new annotation

    klf 12/17/2012

    1. Using the Scrum-dog EI, Genotypes module, search for Acsl4<tm1a(EUCOMM)Wtsi>, Indeterminate genotype
    2. Click on the MP Annotations module button
    3. Add a new annoation MP:0008888 with a normal Qualifier and press Save Annotation
    4. Verify the Qualifier for MP:0008888 is now normal
  2. Verify you can modify an existing HTMP-load annotation sub term with a normal qualifier

    klf 12/17/2012

    1. Using the Scrum-dog EI, Genotypes module, search for Acsl4<tm1a(EUCOMM)Wtsi>, Indeterminate genotype
    2. Click on the MP Annotations module button
    3. Change the Qualifier for MP:0008856 to normal and press Save Annotation
    4. Verify the Qualifier for MP:0008856 is now normal
  3. Verify you can modify an existing non-HTMP load sub term with a normal qualifier

    klf 12/17/2012

    1. Using the Scrum-dog EI, Genotypes module, search for Mitf<Mi-Or>, Heterozygous genotype
    2. Click on the MP Annotations module button
    3. Change the Qualifier for MP:0005172 to normal and press Save Annotation
    4. Verify the Qualifier for MP:0005172 is now normal
  4. Verify you can remove an existing normal qualifier from a non-load annotation term

    klf 12/17/2012

    1. Using the Scrum-dog EI, Genotypes module, search for Il13<tm1(YFP/cre)Lky>, Indeterminate genotype, strain=C.Cg-Il13<tm1(YFP/cre)Lky> (MGI:4461201)
    2. Click on the MP Annotations module button
    3. Change the Qualifier for MP:0003162 from normal to none and press Save Annotation
    4. Verify the Qualifier for MP:0003162 is now none
    5. Return annotation for MP:0003162 to normal again so this data appears in the WI as it was originally entered.
  5. Verify you can remove an existing normal qualifier from a HTMP load annotation term

    2. Click on the MP Annotations module button
    3. Change the Qualifier for MP:0008856 from normal to none and press Save Annotation
    4. Verify the Qualifier for MP:0008856 is now none
  6. Verify all header terms default to normal qualifier

    klf 12/17/2012

    1. Using the Scrum-dog EI, Genotypes module, search for Acsl4<tm1a(EUCOMM)Wtsi>, Indeterminate genotype, strain=Not Specified.
    2. Click on the MP Annotations module button.
    3. Enter Term Acc ID MP:0003631 and tab out of the Term Acc ID field.
    4. Verify MP:0003631 has a qualifier of normal.
    5. Enter Term Acc ID MP:0005377 and tab out of the Term Acc ID field.
    6. Verify MP:0005377 has a qualifier of normal.
    7. Enter Term Acc ID MP:0005384 and tab out of the Term Acc ID field.
    8. Verify MP:0005384 has a qualifier of normal.
    9. Keep these data displayed for the next test...
  7. Confirm you can't change a header term to null qualifier

    klf 12/17/2012

    1. Using the data above...
    2. Change the Qualifier for Term Acc ID MP:0005384 to none and click Save Annotations.
    3. Verify the qualifier does not change but stays as normal, You get the warning "Qualifier 'norm' is the default for MP header terms.", and the change cannot be saved.
