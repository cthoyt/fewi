Definitions

One-Marker Genotype (OMG)
Only 1 allele pair is present in the genotype and both are alleles of the same marker.
May include hom, het+wt or "trans-het" (two different alleles of the same marker).
Simple Genotype
A genotype which can be resolved to the allele pair of one "causal" marker.
A genotype which, after ignoring or removing Recombinase alleles and Reporter alleles, contains only 1 allele pair having both alleles from the same marker.
Complex Genotype
A genotype having two or more allele pairs from two or more markers present.
MGI's "compound" genotypes are treated as "complex" genotypes (in the database, each allele is recorded as an allele pair).
Conditional Genotype
A genotype where the is_conditional flag (set by curators) is true.
These genotypes are complex; they always involve alleles of at least 2 markers.

Logic Summary

Allele Query:

Describes which data we are querying against...

Phenotype/Disease input

  • Exclude genotype-phenotype annotations with qualifier "normal".
  • Exclude genotype-OMIM annotations with qualifier "Not".
  • Include annotations to genotypes having 2 alleles of only 1 marker (OMG).
  • Include annotations to conditional genotypes.
  • Exclude annotations to all other complex genotypes (genotypes having the alleles of 2 or more markers).
  • Include annotations to all kinds of genotypes. (sto115)
  • Input is matched against:
    • MP terms used in annotations
    • MP synonyms
    • MP IDs
    • OMIM terms used in annotations
    • OMIM synonyms
    • OMIM IDs
    • MP annotation notes text:
      • General notes
      • Background Sensitivity notes
      • Normal notes
    • "General" allele notes
    • TEXT-QTL notes
  • Boolean AND, OR, and NOT are supported when entered in uppercase. Otherwise, these words are treated as text strings.
  • Double-quotation marks are treated as phrase queries.
  • Parentheses for grouping text strings are supported. Nested parentheses are allowed.
  • We use the MGI-modified Stop Word list from the Quick Search.
  • Non-alpha-numeric characters are ignored, delimiting tokens.
  • Text string tokens are ORed together unless the user specifies otherwise (enters Boolean ANDs, NOTs, parentheses, etc.)
  • Token order matters only if the user enters double-quotation marks to indicate a phrase query.

Location input

  • Multiselect is supported for chromosomes.
  • Chromosome selections are combined using OR.
  • Dash or hyphen character must separate values in cM position and Genome Coordinates input fields.
  • Inputs are combined using AND.

Allele Summary display:

Describes which data are displayed...

  • Display a row for each non-Wild Type allele for all returned genotypes (including Gt(ROSA)26Sor alleles).
  • Do not display phenotype headers and disease terms where allele is in non-Conditional Complex genotypes.
  • Display phenotype headers and disease terms where:
    • allele is in OMG
    • allele is in Conditional genotypes (includes recombinase alleles, transgenes, and any other kinds of alleles)
  • When allele is annotated to term "No phenotypic analysis" in OMG or Conditional genotype, and no other terms in ANY genotypes, display "No Phenotypic Analysis".
  • When allele is annotated to term "Normal phenotype" or its children in OMG or Conditional genotype, and no other terms in ANY genotypes, display "No Abnormal Phenotype Observed".
  • When allele has "N" annotations (annotated to header terms) in OMG or Conditional genotype, and no other annotated terms in ANY genotypes, display "No Abnormal Phenotype Observed".
  • Display phenotype headers and disease terms for all OMG and Conditional genotypes of an allele on the summary. Do not limit the display to data associated with only those genotypes brought back by the query.
    (I.e., if a query matched data associated with 1 gentoype, but the allele participates in 10 genotypes, also display data associated with the other 9 genotypes.)

Allele Summary Downloadable Reports

We must manipulate some data before sending it to the report file.

  • For Allele Synonyms column, use pipe character as list items delimiter.
  • For Abnormal Phenotypes Reported in these Systems column, use pipe character as list items delimiter.
  • For Human Disease Models column, add OMIM ID and use pipe character as list items delimiter.
